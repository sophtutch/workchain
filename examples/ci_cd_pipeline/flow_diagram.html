Fast Sweep
Discovers workflows with next_poll_at ≤ now or status = pending. Drives the claim-poll-release cycle for async steps.
every 1s → find_claimable()
Heartbeat
Extends lock TTL on all workflows claimed by this instance. If the process dies, the lock expires after TTL seconds.
every TTL/3 → heartbeat()
Slow Sweep
Detects anomalies: steps stuck in SUBMITTED or RUNNING without a lock; completed steps with un-advanced index; stale locks past TTL.
every 30s → find_anomalies()
Fence Token
Incremented on every try_claim(). Every write includes the token in its query filter. If the token doesn’t match, the write is silently rejected — the instance lost its lock.
Instance A claims → fence=N
Lock expires, B claims → fence=N+1
A tries to write → rejected

Retry Failure Path
All Retries Exhausted lock released
When all max_attempts fail, the step is marked FAILED.
Attempt N / N — final attempt fails
↓
steps[i].status → "failed"
workflow.status → "failed"
Lock released. Error + traceback persisted to steps[i].error.
Crash Recovery Path
Crash During Step Execution
Process dies mid-step. Lock TTL expires. Slow sweep detects orphaned step in SUBMITTED or RUNNING state without a live lock.
Recovery Decision Tree
1. Has verify_completion hook?
YES → Call it. If returns True → mark COMPLETED, advance.
NO ↓
2. Is async step with completeness_check?
YES → Run completeness_check. If complete → COMPLETED. Else → set BLOCKED, resume polling.
NO ↓
3. Is step handler idempotent?
YES → Re-execute handler from scratch. Safe because idempotent.
NO → Mark step NEEDS_REVIEW. Human intervention required.
State Transitions
FromToTrigger
pending running try_claim() — atomic findOneAndUpdate, increments fence_token
running running Step completes, dependent steps become ready for claiming
running blocked Async step submitted, lock released, next_poll_at persisted
blocked running completeness_check returns complete, lock kept, step advances
running completed Final step result persisted, all steps done, lock released
running failed All retry attempts exhausted, exception persisted, lock released
running needs_review Crash recovery: step not idempotent and cannot be safely re-run
